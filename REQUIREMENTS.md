# 需求文档 - 播客转文章工具

## 一、项目背景

用户需要将小宇宙播客内容转换为适合微信公众号或小红书发布的文章格式。

## 二、核心需求

### 2.1 内容获取策略（优先级排序）

**需求描述**: 以播客内容为主，shownotes为参考，三级递进获取内容

**实现方案**:
1. **Article模式** - 搜索外部平台是否有现成文稿
   - 搜索微信公众号文章
   - 展示文章列表供选择
   - 获取失败时列出前3个文章链接

2. **Audio模式** - 音频转录（内容最准确）
   - 下载播客音频
   - 使用Whisper模型转录
   - 显示实时进度（下载中→转录中→完成）

3. **Shownotes模式** - 节目简介（保底方案）
   - 爬取小宇宙页面shownotes
   - 当其他模式失败时使用

### 2.2 实时进度显示

**需求描述**: 用户需要看到当前执行的模式和状态

**实现方案**:
- 任务状态字典：`{status, progress, message, mode, failed_articles}`
- 前端轮询：`/api/task/<task_id>` 每2秒查询一次
- 显示内容:
  - 当前执行模式（Article/Audio/Shownotes）
  - 进度百分比
  - 状态消息
  - Article模式失败时的文章链接列表

### 2.3 缓存机制

**需求描述**: 避免重复转录同一播客

**实现方案**:
- 缓存键：URL的MD5哈希
- 存储位置：`web/cache/<md5>.json`
- 缓存内容：转录结果 + 创建时间
- 检查时机：转录任务开始时先查缓存

### 2.4 AI智能生成

**需求描述**: 使用AI将口语化转录内容改写成流畅文章

**实现方案**:
- 支持Kimi (moonshot) 和 Claude API
- 自动检测环境变量选择提供商
- 第一人称→第三人称转换
- 去除口头禅、优化排版
- 失败时回退到模拟生成

**Prompt设计**:
- 角色设定：专业内容编辑
- 叙述视角：以"叙述者"作为人称转换目标
- 结构要求：标题→开头→正文→结尾
- 排版格式：首行缩进、行距、小标题

### 2.5 风格定制

**微信公众号风格参数**:
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| narrator | string | "我的朋友" | 叙述者人称 |
| first_line_indent | boolean | true | 首行缩进 |
| line_height | float | 2.0 | 行距倍数 |
| font_size | int | 16 | 正文字号(px) |

**小红书风格参数**:
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| narrator | string | "我的朋友" | 叙述者人称 |
| use_emoji | boolean | true | 使用Emoji |
| bullet_points | boolean | true | 要点分点 |
| highlight_key_points | boolean | true | 高亮重点 |
| tags | string | "" | 文末标签 |

### 2.6 富文本编辑器

**需求描述**: 在预览页支持富文本编辑

**功能列表**:
- [x] 粗体（`**文本**` → `<strong>`）
- [x] 斜体（`*文本*` → `<em>`）
- [x] 下划线（`<u>文本</u>`）
- [x] 文字颜色（8种颜色选择器）
- [x] 标题（`## `）
- [x] 列表（`- `）

**技术实现**:
- Markdown → HTML 转换函数
- 实时预览同步更新
- 手机/PC双视图切换

### 2.7 预览页面

**需求描述**: 编辑内容时实时预览最终效果

**实现方案**:
- 左右布局：左侧编辑器，右侧预览
- 设备切换：手机/PC视图
- 风格应用：根据用户设置应用字体、行距等
- Markdown渲染：支持粗体、斜体、颜色、下划线

## 三、技术架构

### 3.1 后端
- **框架**: Flask + Flask-CORS
- **并发**: 后台线程处理长任务
- **存储**: 本地文件缓存

### 3.2 前端
- **模板**: Jinja2 + 原生HTML/CSS/JS
- **状态管理**: localStorage
- **通信**: REST API + 轮询

### 3.3 外部服务
- **Kimi API**: OpenAI兼容接口
  - Base URL: `https://api.moonshot.cn/v1`
  - Model: `kimi-k2.5`
  - 限制: temperature只能为1
- **Whisper**: 本地音频转录

## 四、关键决策记录

| 日期 | 决策 | 原因 |
|------|------|------|

| 2025-04-22 | 使用后台线程而非Celery | 简化部署，无需额外依赖 |
| 2025-04-22 | Markdown语法+HTML混合 | 简单直观，易于实现 |
| 2025-04-22 | localStorage保存中间状态 | 页面刷新不丢失进度 |
